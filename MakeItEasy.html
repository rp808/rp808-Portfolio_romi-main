MakeItEasy - Mobile app


  • Role: Product Owner
  • Time: 4 months
  • User interation : Browsing, filtering, rating , commenting, saving the recipe to their profile
  • Technologies: React, React Native, MongoDB, Express.js, TypeScript, CSS3

  • Users: International students studying abroad Single parents with limited time and resources, focused on minimizing food waste and creating new meals from leftovers
  • Goal: The goal of the MakeItEasy mobile app is to simplify meal planning and preparation for users, particularly international students and single parents, by providing tailored recipe suggestions based on available ingredients and dietary preferences. The app aims to help users discover new and creative meal ideas, minimize food waste, and make the most of their limited resources and time constraints.
  • Problem Statement: International students and single parents face challenges in meal planning and preparation due to limited access to ingredients, time constraints, and the need to minimize food waste. They often struggle to find diverse and satisfying meal options using the ingredients they have on hand. Additionally, navigating through various recipe sources can be overwhelming and time-consuming. Therefore, there is a need for a solution that provides personalized recipe recommendations based on available ingredients and dietary preferences, simplifying the meal planning process and helping users make the most of their resources while minimizing food waste.

  • User Stories And Use Cases:
    • User Story:
      As a <beginner user> I want to be able to <create a new account with my name, email, and password> so that I can <access the app>.
      Use Case:
      1. App provides a form for users to create an account(Name, email, password).
      2. User provides the information.
      3. App allows the user to use the application.
    • User Story:
      As a <registered user> I want to be able to <sign in into the MakeItEasy app with my email id and password> so that I can <able to use the benefit of the MakeItEasy app>.
      Use Case:
      1. App provides the login form where the user can enter their registered email and password.
      2. User provides the information.
      3. Once the user enters the required credentials, the app verifies their information.
      4. If the data matches with the records, the app allows the user to access their account.
    • User Story:
      As a <beginner user> I want to be able to <browse random recipes in the app> so that I can <discover new and interesting meal ideas>.
      Use Cases:
      1. The homepage of the app provides different types of recipes with titles and photos for the user.
      2. Users can scroll through the recipe.
    • User Story:
      As a <beginner user> I want to be able to <filter recipes based on the available ingredients and serving size> so that I can choose <from the suggested recipes by the app>.
      Use Cases:
      1. App allows users to navigate to the filter page.
      2. App provides the form in which users can enter the number of serving and available ingredients with quantity.
      3. User provides the information and submit the filter request.
      4. The app analyses the filtered ingredients and suggests recipes that match the ingredients provided by the user.
      5. Users can choose recipes from the listed suggestions.
    • User Story:
      As a <beginner user> I want to be able to <select the recipe and view its detailed information> so that I can <follow the instructions to prepare the meal>.
      Use Cases:
      1. App allows users to select the recipe of their choice.
      2. After the user's selection, the app will navigate to the recipe page which contains a list of required ingredients and instructions to make the meal.
    • User Story:
      As a <beginner user> I want to be able to <remove an ingredient from the list of entered ingredients> so that I can <easily make adjustments to the ingredients>.
      Use Cases:
      1. User opens the recipe app and navigates to the ingredient entry section.
      2. User enters the required ingredients for their recipe.
      3. User notices an ingredient that needs to be removed.
      4. User clicks on the “remove (-)” button.
      5. System will show an option next to each ingredient.
      6. Users will remove the ingredient that they want to remove.
      7. The updated ingredient list is displayed to the user.
    • User Story:
      As a <beginner user> I want to be able to <rate the recipe> so that I can <provide feedback and share my opinion on the recipe’s quality>.
      Use Case:
      1. Users open the recipe page for a specific recipe.
      2. Users scroll down to the rating section on the recipe page.
      3. User selects the desired number of stars for the recipe (ranging from 1 to 5 stars).
      4. The app updates the rating in real-time as the user selects the stars.
      5. The app registers the user’s rating and associates it with the respective recipe.
